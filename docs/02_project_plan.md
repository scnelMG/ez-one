# 02. 프로젝트 기획서

기준 원본: Notion `02. 프로젝트 기획서`

## 제품 개요

EZ-ONE은 여러 공고를 동시에 준비하는 취업준비생을 위한 지원 준비 워크스페이스다. 사용자는 관심 공고를 저장하고, 공고별 워크스페이스에서 자소서 작성, 참고자료 정리, 서류 입력 정보 재사용을 한 흐름으로 처리한다.

## 제품 결정

| 항목 | 결정 |
| --- | --- |
| 제품명 | EZ-ONE |
| 대상 사용자 | 취업준비생, SSAFY 교육생, 반복 지원자 |
| 핵심 문제 | 공고 정보, 작성 맥락, 참고자료, 재사용 가능한 개인정보가 여러 도구에 분산된다. |
| 제품 목표 | 공고 저장부터 지원 워크스페이스, 참고자료, 자소서 작성, 서류 입력 정보 재사용까지 연결한다. |
| 핵심 흐름 | `공고 저장 -> 장바구니 -> 지원 워크스페이스 -> 참고자료/자소서 작성 -> 서류 입력 정보 재사용` |

## MVP 범위

| 영역 | 포함 범위 |
| --- | --- |
| 인증 | Google 로그인, 사용자별 데이터 분리 |
| 온보딩 | 희망 직무, 기업 유형, 산업, 지역, 기술스택, SSAFY 여부 저장 |
| 대시보드 | 상태 카드에서 장바구니 필터/정렬로 이동 |
| 공고 장바구니 | 확장 프로그램, 추천 공고, 직접 입력으로 공고 저장 |
| 워크스페이스 | 저장 공고별 워크스페이스 자동 생성 |
| 작성 | 도화지, 글자 수, 자동 저장 상태, 명시적 자소서 버전 생성/비교 |
| 참고자료 | 자유 메모, 커스텀 보드, JD/news/DART/인재상 수동 입력 |
| 서류 입력 정보 | 기본정보, 학력, 경력, 프로젝트, 자격/어학, 수상/활동, 커스텀 항목 |
| Chrome Extension | Jasoseol.com 공고 추출, 미리보기, 저장 |
| 추천 | Jasoseol.com 기반 추천 공고 별표 저장 |
| Notion | 저장 공고 `JOB_ONLY` 동기화 |

## MVP 제외 범위

| 영역 | 제외 기준 |
| --- | --- |
| 알림 | P1은 화면 내 상태 표시만 제공한다. 독립 알림 채널은 후순위다. |
| 과거 지원 내역 | IA에는 유지하되 MVP 구현 범위에서는 제외한다. |
| Mattermost 수집 | SSAFY 추천 구조는 남기되 raw 수집과 후보화는 후순위다. |
| 자동 참고자료 수집 | P1은 JD/news/DART/인재상을 사용자가 직접 입력한다. |
| 서류 자동 입력 보조 | 공고 저장 흐름 안정화 이후 확장한다. |
| Notion 확장 동기화 | 자소서/도화지 동기화는 P2다. P1 scope는 `JOB_ONLY`다. |
| 관리자 화면 | 초기 사용자 MVP에서 제외한다. |

## 완료 기준

| 기준 | 통과 조건 |
| --- | --- |
| 공고 저장 | P1 저장 경로로 저장한 공고가 장바구니에 표시되고 워크스페이스를 가진다. |
| 다음 행동 판단 | 대시보드 카드와 장바구니 필터로 상태, 마감, 준비 단계를 확인한다. |
| 작성 맥락 유지 | 워크스페이스에서 공고 정보, 참고자료, 도화지, 자소서 버전을 함께 접근한다. |
| 재사용 데이터 | 서류 입력 정보가 저장되고 워크스페이스 기본값 또는 입력 보조 데이터로 재사용된다. |
| 외부 실패 격리 | Notion/Extension 실패가 핵심 DB 저장 데이터를 훼손하지 않는다. |

## 구현 원칙

- P1/P2/P3 범위는 `docs/04_requirements.md`를 우선한다.
- 요구사항 연결 검증은 `docs/23_traceability.md`에서 관리한다.
- 사용자 소유 데이터는 모든 조회, 수정, 삭제에서 ownership을 검증한다.
- 외부 연동 실패는 핵심 DB 저장 트랜잭션과 분리한다.
- 초기 기술 스택은 Spring Boot, Spring MVC, Spring Security, JWT, MyBatis, Vue 3, Vite, Chrome Extension, MySQL, Google OAuth2, Notion API, AWS EC2를 기준으로 한다. DB migration 도구는 보류한다.
