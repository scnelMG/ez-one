# 06. 정책 정의서

Source of truth: Notion `06. 정책 정의서`

## 공고 / 장바구니

| 정책 | 기준 |
| --- | --- |
| 장바구니 의미 | 사용자가 관심 있어 저장한 공고 목록 |
| 저장 경로 | Chrome Extension, 추천 공고, 직접 입력 |
| 중복 기준 | P1은 URL을 우선 중복 키로 사용한다. |
| 워크스페이스 생성 | 공고 저장 시 자동 생성한다. |
| 삭제 | 사용자 소유 데이터는 soft delete/archive를 우선한다. |

## 지원 상태

| 상태 | 의미 | 처리 기준 |
| --- | --- | --- |
| `READY` | 저장됐지만 작업을 시작하지 않음 | 공고 저장 기본값 |
| `IN_PROGRESS` | 자소서, 참고자료, 지원 정보 작업 시작 | 워크스페이스 작업 시 수동 또는 자동 전환 가능 |
| `COMPLETED` | 사용자가 지원 완료로 표시 | 수동 변경 |
| `NOT_APPLIED` | 마감 경과 또는 포기 | 완료 상태가 아니면 마감 후 자동 전환 가능 |

## 워크스페이스

| 정책 | 기준 |
| --- | --- |
| 자동 저장 | UI는 즉시 dirty 상태를 표시하고, API 저장은 1초 debounce와 최대 5초 forced save를 사용한다. |
| 최신 초안 | 자동 저장은 최신 draft row를 갱신한다. 입력마다 version row를 만들지 않는다. |
| 버전 생성 | 비교 가능한 버전은 사용자가 명시적으로 저장할 때 생성한다. |
| 버전 비교 | 한 번에 두 버전만 비교한다. |
| 참고자료 | P1은 자유 메모/커스텀 보드와 JD/news/DART/인재상 수동 입력을 지원한다. |
| 참고자료 열람 | 제목 클릭은 전체 페이지, 보조 버튼은 사이드 패널을 연다. |
| 자동 참고자료 수집 | P2 범위다. |

## 외부 연동

| 정책 | 기준 |
| --- | --- |
| Extension 수집 | 사용자가 지원 사이트에서 확장 프로그램을 실행한 경우에만 수집한다. |
| P1 지원 사이트 | Jasoseol.com 공고 저장부터 시작한다. |
| 추천 소스 | P1은 Jasoseol.com 기반 추천을 사용한다. |
| Mattermost | SSAFY 추천 구조는 유지하되 raw 수집/후보화는 P2다. |
| 기업 정보 | P1에서 자동 수집하지 않는다. 필요한 값은 비워둘 수 있다. |
| 외부 실패 | Notion, Mattermost, 추천 수집 실패는 핵심 DB 저장 트랜잭션과 분리한다. |

## Notion

| 정책 | 기준 |
| --- | --- |
| P1 sync | 저장 공고만 `JOB_ONLY` 자동 동기화한다. |
| Sync scope | `JOB_WITH_ESSAY`, `JOB_WITH_ESSAY_AND_CANVAS`는 P2다. |
| 계정 구분 | 서비스 로그인 계정과 Notion 연결 계정은 다를 수 있음을 표시한다. |
| 구조 변경 | 연결된 Notion DB/page 구조가 바뀌면 경고한다. |
| 수동 export | 보조 기능이며 job-only 자동 동기화를 대체하지 않는다. |

## 보안 / 개인정보

- 필요한 개인정보만 수집하고 수정/삭제 가능해야 한다.
- 사용자 소유 데이터는 항상 ownership을 검증한다.
- 실제 토큰, 비밀값, 원본 개인정보를 로그, 문서, 코드에 남기지 않는다.
- API 응답은 DB row/entity를 직접 노출하지 않고 DTO로 반환한다.
