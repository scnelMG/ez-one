## 다이어그램 참고사항

- GitHub/Notion Mermaid 호환을 위해 `flowchart` 문법으로 UML 유즈케이스 다이어그램 형태를 표현한다.
- 사각형은 액터/외부 시스템, 큰 박스는 EZ One 시스템 경계, 타원은 유즈케이스를 뜻한다.
- P1 구현 판단은 [04. 요구사항 정의서](04_requirements.md)와 [23. 요구사항 추적표](23_traceability.md)를 우선한다.
- UC-13, UC-14, UC-16은 IA에는 유지하지만 P1 완료 기준이 아니다.
- 아래 UC 상세 명세는 API, 화면, 테스트 설계 시 흐름 기준으로 사용한다.

## UC-01. Google 로그인

| 항목 | 내용 |
| --- | --- |
| 시스템 제목 | EZ-ONE MVP |
| 유즈케이스 이름 | Google 로그인 |
| 액터 | 사용자, Google OAuth |
| 시작 조건 | 사용자가 EZ-ONE 웹 서비스에 접속해 있어야 한다. |
| 기본 흐름 | 1. 사용자가 Google 로그인 버튼을 선택한다.<br>2. 시스템이 Google OAuth 인증 화면을 요청한다.<br>3. 사용자가 Google 계정으로 인증한다.<br>4. Google OAuth가 인증 결과를 시스템에 전달한다.<br>5. 시스템이 사용자 계정 정보를 확인한다.<br>6. 최초 로그인 여부를 확인한다.<br>7. 최초 로그인 사용자는 온보딩 모달로 이동한다.<br>8. 기존 사용자는 메인 페이지로 이동한다. |
| 대안 흐름 | 3A. 사용자가 Google 인증을 취소한다.<br>1. 시스템이 로그인 전 상태로 돌아간다.<br><br>4A. Google 인증에 실패한다.<br>1. 시스템이 로그인 실패 메시지를 표시한다.<br>2. 사용자는 다시 로그인을 시도할 수 있다. |
| 종료 조건 | 사용자가 로그인 상태가 되거나, 로그인 실패/취소 상태로 남는다. |

## UC-02. 온보딩 입력 또는 건너뛰기

| 항목 | 내용 |
| --- | --- |
| 시스템 제목 | EZ-ONE MVP |
| 유즈케이스 이름 | 온보딩 입력 또는 건너뛰기 |
| 액터 | 사용자 |
| 시작 조건 | 사용자가 최초 로그인 상태여야 한다. |
| 기본 흐름 | 1. 시스템이 온보딩 모달을 표시한다.<br>2. 사용자가 희망 직무를 입력한다.<br>3. 사용자가 희망 기업, 희망 지역, 보유 스킬을 입력한다.<br>4. 사용자가 SSAFY 교육생 여부를 선택한다.<br>5. 사용자가 시작하기 버튼을 선택한다.<br>6. 시스템이 온보딩 정보를 저장한다.<br>7. 시스템이 사용자를 메인 페이지로 이동시킨다. |
| 대안 흐름 | 1A. 사용자가 온보딩을 건너뛴다.<br>1. 시스템이 온보딩 정보를 저장하지 않는다.<br>2. 시스템이 사용자를 메인 페이지로 이동시킨다.<br>3. 사용자는 이후 마이페이지에서 온보딩 정보를 입력하거나 수정할 수 있다. |
| 종료 조건 | 사용자가 메인 페이지에 진입한다. 온보딩을 입력한 경우 해당 정보가 마이페이지에 저장된다. |

## UC-04. 대시보드 항목 클릭으로 공고 장바구니 필터링/정렬

| 항목 | 내용 |
| --- | --- |
| 시스템 제목 | EZ-ONE MVP |
| 유즈케이스 이름 | 대시보드 항목 클릭으로 공고 장바구니 필터링/정렬 |
| 액터 | 사용자 |
| 시작 조건 | 사용자가 메인 페이지에 있고, 공고 장바구니에 공고 데이터가 존재해야 한다. |
| 기본 흐름 | 1. 사용자가 메인 페이지의 대시보드를 확인한다.<br>2. 사용자가 대시보드 항목 중 하나를 선택한다.<br>3. 시스템이 공고 장바구니 페이지로 이동한다.<br>4. 시스템이 선택한 항목에 맞게 공고 목록을 필터링하거나 정렬한다.<br>5. 시스템이 조건이 반영된 공고 장바구니 목록을 표시한다. |
| 대안 흐름 | 2A. 사용자가 총 지원 항목을 선택한다.<br>1. 시스템이 전체 공고 목록을 표시한다.<br><br>2B. 사용자가 마감 임박 항목을 선택한다.<br>1. 시스템이 공고 목록을 마감 임박순으로 정렬한다.<br><br>2C. 사용자가 진행중 항목을 선택한다.<br>1. 시스템이 지원 상태가 진행중인 공고만 표시한다.<br><br>2D. 사용자가 지원 전 항목을 선택한다.<br>1. 시스템이 지원 상태가 지원 전인 공고만 표시한다. |
| 종료 조건 | 공고 장바구니가 선택한 조건에 맞게 표시된다. |

| 종료 조건 | 사용자가 추천 공고를 확인하거나, 관심 공고가 공고 장바구니에 저장된다. |

## UC-06. 확장 프로그램으로 공고 저장

| 항목 | 내용 |
| --- | --- |
| 시스템 제목 | EZ-ONE MVP |
| 유즈케이스 이름 | 확장 프로그램으로 공고 저장 |
| 액터 | 사용자, 브라우저 확장 프로그램, 외부 채용 공고 페이지 |
| 시작 조건 | 사용자가 로그인되어 있고 브라우저 확장 프로그램을 사용할 수 있어야 한다. |
| 기본 흐름 | 1. 사용자가 외부 채용 공고 페이지에 접속한다.<br>2. 사용자가 브라우저 확장 프로그램을 실행한다.<br>3. 사용자가 공고 저장하기 버튼을 선택한다.<br>4. 확장 프로그램이 현재 페이지의 공고 정보를 읽어온다.<br>5. 확장 프로그램이 공고 내 직무 정보를 확인한다.<br>6. 공고에 여러 직무가 있으면 직무 목록을 체크박스로 표시한다.<br>7. 사용자가 저장할 직무를 하나 이상 선택한다.<br>8. 사용자가 저장을 실행한다.<br>9. 시스템이 선택한 직무별 공고를 공고 장바구니에 저장한다. |
| 대안 흐름 | 5A. 공고에 직무가 하나만 있다.<br>1. 시스템이 직무 선택 단계를 생략하거나 단일 직무를 기본 선택한다.<br><br>4A. 공고 정보를 읽어오지 못한다.<br>1. 시스템이 공고 정보 감지 실패 메시지를 표시한다.<br>2. 사용자는 다시 시도하거나 수동 입력을 고려한다. |
| 종료 조건 | 선택한 공고가 공고 장바구니에 저장된다. |
