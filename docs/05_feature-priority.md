# 05. 기능 우선순위 선정 문서

Source of truth: Notion `05. 기능 우선순위 선정 문서`

이 문서는 구현 순서를 정하기 위한 기준이다. 기능 범위가 충돌하면 `docs/04_requirements.md`를 우선한다.

## 우선순위 기준

| 우선순위 | 기준 |
| --- | --- |
| Must | MVP 핵심 흐름에 반드시 필요하다. |
| Should | 핵심 흐름을 강화하지만 Must 이후 구현해도 된다. |
| Could | 데이터 축적 또는 외부 연동 안정화 후 구현한다. |
| Won't for MVP | 초기 구현 범위에서 제외한다. |

## Must

| 기능 | 요구사항 | 이유 |
| --- | --- | --- |
| Google 로그인 | `AUTH-001` | 사용자별 데이터 분리 |
| 온보딩 프로필 | `ONB-001`, `ONB-002` | 추천과 개인화 기준 |
| 공고 장바구니 | `JOB-001` | 저장 공고의 중심 목록 |
| 워크스페이스 자동 생성 | `JOB-002` | 공고별 작성 공간 생성 |
| Chrome Extension 공고 저장 | `EXT-001` | Jasoseol.com 공고 저장부터 시작 |
| 대시보드 요약/라우팅 | `DASH-001` | 상태 카드가 장바구니 필터/정렬로 이동 |
| 서류 입력 정보 | `PROFILE-001` | 지원서 입력 보조용 재사용 데이터 |
| 워크스페이스 작성 | `WS-001`, `WS-002`, `WS-003` | 도화지, 자동 저장, 기본값 입력 |
| 자소서 버전관리 | `WS-004` | 명시적 버전 생성과 비교 |
| 수동 참고자료 | `REF-001`, `REF-002` | 자유 메모, 커스텀 보드, JD/news/DART/인재상 수동 입력 |
| 추천 공고 저장 | `REC-001` | Jasoseol.com 기반 추천과 star-to-basket |
| Notion job-only sync | `NOTION-001` | 저장 공고만 자동 동기화 |

## Should

| 기능 | 이유 |
| --- | --- |
| 마이페이지 정보 수정 | 온보딩/서류 입력 정보 수정 경로 |
| 프롬프트 저장/복사 | 작성 워크플로우 보조 |
| 확장 프로그램 서류 입력 보조 | 공고 저장 이후 확장 기능 |
| Mattermost 수집/후보화 | SSAFY 추천 고도화 소스 |
| Notion 계정/schema 경고 | job-only 동기화 안정화 이후 필요 |

## Could

| 기능 | 이유 |
| --- | --- |
| 추천 hover 기업 정보 | 기업 데이터 안정화 후 |
| 과거 지원 통계 | 데이터 누적 후 |
| 장바구니 캘린더/주간 일정 | 장바구니 P1 목록/정렬 완성 후 마감 일정 시각화로 검토 |
| 자동 JD/news/DART/인재상 수집 | 수동 입력 흐름 완성 후 |
| 기업 정보 자동 수집 | 외부 사이트 정책 검토 후 |
| AI 요약/평가 | MVP 이후 별도 검토 |

## Won't for MVP

- 관리자 운영 화면
- 별도 AI 서버
- P1 자동 참고자료 수집
- P1 자소서/도화지 Notion 동기화
- P1 독립 알림 채널
- P1 고객지원/문의 운영 화면
